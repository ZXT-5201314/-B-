# GPM_3IMERGHHE.07 数据介绍

本文档面向参赛者介绍数据包 `GPM_3IMERGHHE.07` 的基本情况、文件格式与目录组织方式，便于快速读取和使用数据。

## 数据概览

- 数据名称：`GPM_3IMERGHHE.07`
- 数据时间范围：压缩包内样本覆盖 `2014070` 至 `2024270`
- 首个观测文件时间：`2014-06-09 00:00`
- 最后一个观测文件时间：`2024-11-16 12:00`
- 数据组织规模：共 `110` 个事件目录、`30000多` 个 GeoTIFF 文件
- 数据来源类型：GPM IMERG 半小时降水栅格快照
- 降水值是半小时的累计降水量，单位是毫米/时(mm/hr)。这里的“mm/hr”表示如果按照每小时来衡量，半小时内积累的降水量是多少毫米。例如，如果该变量值为10mm/hr，那么在这半小时内累计的降水量就是5mm（半小时的累计量).
可以将本数据理解为：以台风事件为单位组织的半小时降水快照集合。每个子目录对应一个台风事件，每个 `.tif` 文件对应该事件某一时次的降水空间分布。

## 文件格式说明

本数据包中的核心数据文件为 `.tif` 格式栅格文件，即 GeoTIFF 文件。对样本文件的实际检查结果表明：

- 文件类型：单波段 GeoTIFF
- 数据类型：`float32`
- 坐标参考系：`EPSG:4326`
- 栅格大小：`201 × 201`
- 空间分辨率：`0.1° × 0.1°`
- 缺测值：`-9999`

因此，参赛者可以把每个 `.tif` 文件理解为一个二维降水矩阵，同时带有明确的地理参考信息。Matlab和Python下.tif文件的读取，请网上自行查阅相关资料。读取后通常可得到：

- 一个 `201 × 201` 的降水数组
- 对应的经纬度范围
- 对应时次的空间位置与台风中心信息

典型文件名如下：

```text
3B-HHR-E.MS.MRG.3IMERG.20241222-S120000-E122959.0720.V07B_center_113.90E_10.20N_bbox_103.85E_123.95E_0.15N_20.25N.tif
```

文件名中包含以下关键信息：

- `20241222`：观测日期
- `S120000-E122959`：该文件对应的时间窗口，表示 `12:00:00` 到 `12:29:59`
- `V07B`：产品版本号
- `center_113.90E_10.20N`：该时次对应的台风中心经纬度
- `bbox_103.85E_123.95E_0.15N_20.25N`：该栅格覆盖的经纬度范围

根据参考文档，文件时间步长为半小时，因此同一事件目录下的相邻文件通常对应连续的 30 分钟观测时次。

## 文件组织说明

压缩包解压后，数据按“事件目录 + 时次文件”的方式组织。整体结构可表示为：

```text
GPM_3IMERGHHE.07/
  2014010/
    3B-HHR-E.MS.MRG.3IMERG.20140117-S060000-E062959....tif
    3B-HHR-E.MS.MRG.3IMERG.20140117-S063000-E065959....tif
    ...
  2014020/
    ...
  ...
  2024280/
    3B-HHR-E.MS.MRG.3IMERG.20241226-S060000-E062959....tif
```

其中：

- 一级子目录名如 `2014010`、`2024280`，表示单个台风的唯一事件编号
- 一个子目录对应一个完整台风事件
- 一个目录内包含该事件从生成到结束过程中多个半小时时次的降水快照
- 不同事件持续时间不同，因此不同目录中的 `.tif` 文件数量并不相同

这类目录编号可与台风属性表中的事件编号字段对应，用于把台风路径信息与降水文件进行匹配。对于参赛者而言，最直接的使用方式是：

1. 先遍历一级子目录，把每个目录视为一个独立事件样本。
2. 再按文件名中的日期和时间顺序读取目录内 `.tif` 文件，恢复该事件的时间演化过程。
3. 如需构造模型输入，可将单个 `.tif` 作为二维空间样本，或将同一目录下的全部 `.tif` 组成时空序列样本。

## 使用建议

- 若只做单时刻建模，可把每个 `.tif` 视为一张带地理坐标的降水图像。
- 若做时序建模，建议以事件目录为单位组织样本，再按文件名时间排序。
- 读取时应注意将 `-9999` 识别为缺测值，避免直接参与统计计算。
- 若需要配合台风路径或属性数据使用，应通过事件目录名与对应事件编号及相关的时间、位置信息进行关联。

以上说明旨在帮助参赛者快速理解数据结构。
